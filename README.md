# DEV-SAY

## 项目简介

欢迎来到 dev-say！这是一个开源的文档项目，旨在帮助程序员学习和查询常见技术术语、框架、工具和编程语言的正确发音。所有术语按字母顺序组织，提供国际音标和简单描述，便于快速查找和学习。

### 项目目的和价值

- 📚 **系统学习**: 按字母顺序组织的技术术语，覆盖编程语言、框架、数据库、工具等各个领域
- 🎯 **准确发音**: 提供国际音标(IPA)，帮助准确掌握术语读法
- 🤝 **社区共建**: 通过GitHub协作，让每个人都能贡献和完善发音信息
- 🔍 **快速查阅**: 简洁的表格格式和字母索引，便于快速查找

## 使用说明

### 如何查阅 dev-say.md

1. **使用首字母索引**: 文档顶部提供A-Z索引，点击字母快速跳转

2. **按字母顺序浏览**: dev-say.md 按照首字母(A-Z)组织，所有术语在各自字母分组内按字母顺序排列

3. **查看完整信息**: 每个术语包含：
   - 英文原名
   - 国际音标 (IPA)
   - 简单描述

### 如何提交新术语

#### 格式要求

每个术语条目必须包含以下三个字段：

1. **术语名称**: 英文原名（如 `PostgreSQL`）
2. **国际音标**: 使用IPA格式（如 `/ˈpoʊstɡres/`）
3. **简单描述**: 一句话说明该术语是什么（如 `开源关系型数据库`）

#### 提交流程

1. **Fork 本仓库**
   ```bash
   # 点击页面右上角的 Fork 按钮
   ```

2. **克隆到本地**
   ```bash
   git clone https://github.com/Galaxsy/dev-say.git
   cd dev-say
   ```

3. **创建新分支**
   ```bash
   git checkout -b add-new-term
   ```

4. **编辑 dev-say.md**
   - 找到术语首字母对应的分组（如 P 开头的术语在 ## P 分组下）
   - 按字母顺序插入新术语
   - 确保表格格式正确（三列）

5. **提交更改**
   ```bash
   git add dev-say.md
   git commit -m "添加术语: YourTerm"
   git push origin add-new-term
   ```

6. **创建 Pull Request**
   - 访问你的 Fork 仓库
   - 点击 "New Pull Request"
   - 填写 PR 描述，说明添加的术语和原因
   - 等待自动化检查通过
   - 等待维护者审核

#### 多种读法处理

如果一个术语有多种读法，请按照以下顺序列出：
1. 官方推荐读法
2. 常见读法
3. 其他读法

**示例：**

```markdown
| SQL | /ˈɛs kjuː ˈɛl/ 或 /ˈsiːkwəl/ | 结构化查询语言 |
| GIF | /ɡɪf/ 或 /dʒɪf/ | 图像文件格式 |
| LaTeX | /ˈlɑːtɛk/ 或 /ˈleɪtɛk/ | 文档排版系统 |
```

#### 方言差异标注

如果英式和美式发音差异显著，可以标注：

```markdown
| schedule | /ˈskedʒuːl/ (美) 或 /ˈʃedjuːl/ (英) | 调度、时间表 |
| tomato | /təˈmeɪtoʊ/ (美) 或 /təˈmɑːtoʊ/ (英) | 番茄（示例） |
```

#### 常见错误示例

**❌ 错误格式：**
```markdown
| Python | python | 编程语言 |  # 缺少音标斜杠
| React | /riækt | 前端库 |  # 缺少重音符号
| Django | /dʒæŋɡoʊ/ | |  # 缺少简单描述
| Redis |  | 数据库 |  # 缺少国际音标
```

**✅ 正确格式：**
```markdown
| Python | /ˈpaɪθɑn/ | 编程语言 |
| React | /riˈækt/ | 前端库 |
| Django | /ˈdʒæŋɡoʊ/ | Python框架 |
| Redis | /ˈredɪs/ | 数据库 |
```

#### 描述文字规范

简单描述应该：
- ✅ 简洁明了，一句话说明
- ✅ 使用中文
- ✅ 说明术语的类型或用途
- ❌ 不要过长（建议不超过20个字）
- ❌ 不要包含主观评价

**好的描述：**
- "开源关系型数据库"
- "JavaScript运行时环境"
- "容器化应用平台"
- "前端构建工具"

**不好的描述：**
- "一个非常好用的数据库" （主观评价）
- "PostgreSQL是一个功能强大的开源对象关系型数据库管理系统..." （太长）
- "database" （应使用中文）

### 国际音标输入指南

输入国际音标可能比较困难，以下是详细的输入方法和参考资料：

#### 在线IPA键盘工具

1. **[IPA Character Picker](https://ipa.typeit.org/)**
   - 最简单的在线工具
   - 点击符号即可复制
   - 支持所有IPA字符

2. **[IPA Keyboard](https://www.ipachart.com/)**
   - 交互式IPA图表
   - 可以听到每个音素的发音
   - 适合学习和查找符号

3. **[IPA Phonetic Keyboard](https://westonruter.github.io/ipa-chart/keyboard/)**
   - 键盘式布局
   - 支持快捷键输入
   - 适合批量输入

#### 常用IPA符号速查表

**元音 (Vowels)**:
- 前元音: /i/ (see), /ɪ/ (sit), /e/ (say), /ɛ/ (bed), /æ/ (cat)
- 中元音: /ə/ (about), /ʌ/ (cup), /ɜː/ (bird)
- 后元音: /ɑ/ (father), /ɔ/ (law), /o/ (go), /u/ (food), /ʊ/ (book)

**辅音 (Consonants)**:
- 塞音: /p/ (pen), /b/ (bed), /t/ (top), /d/ (dog), /k/ (cat), /ɡ/ (go)
- 擦音: /f/ (fan), /v/ (van), /θ/ (think), /ð/ (this), /s/ (see), /z/ (zoo), /ʃ/ (she), /ʒ/ (vision), /h/ (hat)
- 鼻音: /m/ (man), /n/ (no), /ŋ/ (sing)
- 流音: /l/ (let), /r/ (red)
- 半元音: /w/ (wet), /j/ (yes)

**其他符号**:
- 主重音: /ˈ/ 放在重读音节前 (如 /ˈæpl/ = apple)
- 次重音: /ˌ/ 放在次重读音节前
- 长音: /ː/ 表示元音延长 (如 /iː/ = see)

#### 如何查找术语的正确音标

1. **在线词典**:
   - [Cambridge Dictionary](https://dictionary.cambridge.org/) - 英式和美式发音，带音标
   - [Merriam-Webster](https://www.merriam-webster.com/) - 美式发音权威
   - [Oxford Learner's Dictionaries](https://www.oxfordlearnersdictionaries.com/) - 学习者友好

2. **发音网站**:
   - [Forvo](https://forvo.com/) - 真人发音示例，多种口音
   - [YouGlish](https://youglish.com/) - 在YouTube视频中查找真实发音

3. **官方文档**:
   - 优先查看技术术语的官方网站
   - 很多项目会在官方文档中说明发音
   - 例如: PostgreSQL官网明确说明读作 "post-gres"

#### 音标输入技巧

1. **复制粘贴法** (推荐新手):
   - 使用在线IPA键盘工具
   - 点击符号后复制
   - 粘贴到编辑器中

2. **输入法设置** (适合频繁贡献):
   - macOS: 系统偏好设置 → 键盘 → 输入法 → 添加 "Unicode Hex Input"
   - Windows: 使用 Alt + 数字代码输入
   - Linux: 使用 Compose key 组合

3. **编辑器插件**:
   - VS Code: 安装 "IPA Phonetics" 扩展
   - Sublime Text: 安装 "IPA Input" 插件

#### 音标标注注意事项

1. **使用斜杠包围**: 音标应该用 `/` 包围，如 `/ˈpɑːstɡres/`
2. **标注重音**: 多音节词要标注主重音位置
3. **区分长短音**: 注意 /i/ 和 /iː/ 的区别
4. **参考多个来源**: 如果发音有争议，查找多个权威来源
5. **注明方言差异**: 如果英式和美式发音差异较大，可以都列出

#### 示例对照

| 术语 | 音标 | 说明 |
|------|------|------|
| Python | /ˈpaɪθɑn/ | 主重音在第一音节 |
| JavaScript | /ˈdʒɑːvəskrɪpt/ | 注意长音 /ɑː/ |
| MySQL | /maɪ ˈɛs kjuː ˈɛl/ | 三个独立音节 |
| nginx | /ˌɛndʒɪnˈɛks/ | 次重音和主重音都要标注 |

## 常见问题解答 (FAQ)

### 关于贡献

**Q: 我不确定某个术语的发音是否正确，可以提交吗？**

A: 可以！请在 Pull Request 中说明你的发音来源（如官方文档、词典等），并标注你不确定的地方。维护者和社区成员会帮助审核和讨论。

**Q: 我想添加的术语已经存在了，但发音信息不完整或有误，怎么办？**

A: 很好！请直接修改现有条目，并在 Pull Request 中说明你的修改原因和参考来源。更新现有术语和添加新术语一样重要。

**Q: 一个术语有多种读法，应该如何处理？**

A: 在"国际音标"列中用"或"连接多种读法。优先列出官方推荐的读法，然后是常见读法。例如：
```markdown
| SQL | /ˈɛs kjuː ˈɛl/ 或 /ˈsiːkwəl/ | 结构化查询语言 |
```

**Q: 我不会使用 Git 和 GitHub，还能贡献吗？**

A: 可以！你可以：
1. 在 [Issues](https://github.com/Galaxsy/dev-say/issues) 中提交术语建议，由其他贡献者帮你添加
2. 学习 Git 基础知识，推荐资源：[GitHub 官方教程](https://docs.github.com/cn/get-started)

**Q: 提交 Pull Request 后需要等多久才能被合并？**

A: 通常在 1-3 个工作日内会有维护者审核。如果自动化检查失败，请先修复问题。如果超过一周没有回应，可以在 PR 中评论提醒。

### 关于格式

**Q: 国际音标太难输入了，可以不写吗？**

A: 国际音标是最准确的发音标注方式，是必填字段。如果确实困难，可以：
1. 使用我们提供的[在线IPA键盘工具](#在线ipa键盘工具)
2. 在 Issue 中提交术语建议，由其他贡献者帮你添加音标

**Q: 表格格式总是对不齐，怎么办？**

A: 不用担心！你可以：
1. 使用 Markdown 编辑器（如 Typora、VS Code）的表格格式化功能
2. 只要内容正确，格式问题维护者会帮你修正

**Q: 术语应该放在哪个字母分组下？**

A: 根据术语的首字母确定分组。例如：
- PostgreSQL 放在 ## P 分组下
- React 放在 ## R 分组下
- 在分组内按字母顺序插入术语

### 关于发音

**Q: 英式发音和美式发音差异很大，应该用哪个？**

A: 优先使用美式发音（因为大多数技术术语源自美国）。如果差异显著，可以都列出：
```markdown
| schedule | /ˈskedʒuːl/ (美) 或 /ˈʃedjuːl/ (英) | 调度、时间表 |
```

**Q: 如何确定一个术语的"官方"发音？**

A: 按以下优先级查找：
1. 项目官方网站或文档中的说明
2. 项目创始人或核心开发者的发音视频
3. 权威技术会议或演讲中的发音
4. 英语词典中的标准发音

**Q: 缩写词应该怎么标注发音？**

A: 缩写词通常有两种读法：
1. 字母逐个读：如 SQL = /ˈɛs kjuː ˈɛl/
2. 作为单词读：如 SQL = /ˈsiːkwəl/

如果两种都常见，请都列出。
